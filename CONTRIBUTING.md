# Contributing to SESMailEngine Documentation

Thank you for your interest in improving SESMailEngine documentation!

## Reporting Documentation Issues

If you find errors, unclear explanations, or outdated information in our documentation:

1. **Check existing issues** - Search GitHub Issues to see if it's already reported
2. **Create a new issue** with:
   - The documentation file and section affected
   - What's incorrect or unclear
   - Suggested correction (if you have one)

### Issue Template

```
**Document:** [e.g., docs/INTEGRATION.md]
**Section:** [e.g., Python Examples]
**Issue:** [Describe what's wrong or unclear]
**Suggested Fix:** [Optional - your proposed correction]
```

## Suggesting Improvements

We welcome suggestions for:
- New code examples
- Clarifications to existing content
- Additional troubleshooting scenarios
- Better explanations of complex topics

Create an issue with the label `enhancement` and describe your suggestion.

## Code of Conduct

When participating in this project:

- Be respectful and professional
- Focus on constructive feedback
- Keep discussions relevant to documentation
- Respect that SESMailEngine is a commercial product

## Product Support

For product-related questions (not documentation issues):
- Visit [sesmailengine.com](https://www.sesmailengine.com) for support options
- Check the [Troubleshooting Guide](docs/TROUBLESHOOTING.md) first

## Note

This repository contains documentation only. The SESMailEngine product source code is distributed separately to licensed customers via [sesmailengine.com](https://www.sesmailengine.com).
